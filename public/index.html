REST API Model Documentation

The documentation on the functionality and understanding of how this API works is detailed below.

Goal:

This REST API defines various endpoints for client-side interaction. It provide routes for different functionality such as getting all employees, deleting an employee, creating an employee and updating an employee.



HTTP Method Request URL Request body data format Response body data format
GET Get all employees '/employees' None Will get all data array of all employee lists in JSON format
GET Get data about a single employee '/employees/:id' None Get an object of an employee in JSON format, based on id. Example: { Name: 'Mary', Surname: 'Poppins', Age: '45', id: '313abc8798298eetwewe222445' }
POST Allow employee to create account '/employees' A JSON object holding data about the new employee
Example: { Name: 'Grace', Surname: 'Jake', Age: "24" } 		A JSON format of the employee's added information, included with a unique id and an extra success proof message
Example: { id: 313abc8798298eetwewe222445, username: 'Grace', Surname: 'Jake', Age:"24" }
PUT Allow employee to update their info '/employees/:id' None A JSON format data with the updated result
DELETE Allow employee to delete account '/employees' None Returns a text confirming account deleted
